# 2026-07-14

## Cyphal actuator backend: increment 2c plan (handoff)

Increments 1, 2a, 2b are done and pushed on branch `cyphal-actuator-backend`
(coxswain-cyphal transport crate; the OutputBackend trait + $CXOUT refactor to
the physical-units boundary, D-028; the CyphalActuatorBackend with
command-then-report). 2c is the hosted integration, one coupled unit, to be
done as its own focused session. Four parts:

1. **Effector schema (signed blob, schema_version 4 -> 5).** An effector's
   output wiring depends on its bus kind. Cleanest shape: an `EffectorOutput`
   enum, `Serial { channel, pwm }` vs `Cyphal { node_id, command_subject,
   feedback_subject }`, selected by the effector's bus kind at compile.
   Validation: a `cyphal_can` effector requires the Cyphal fields (not
   channel/pwm); an `actuator_uart`/`pwm` effector requires channel/pwm (not
   subjects). Blast radius: bump schema_version, update every golden/tamper/
   version-rejection test, the manifest schema doc, and the blob reader; add a
   `cyphal_can` effector to a test manifest. Record as a DECISIONS entry
   (D-029, or a D-028 follow-on) BEFORE coding, since it changes the blob
   format (D-018).

2. **Read-write CAN transport (`hosted/src/can.rs`).** Today it is listen-only
   by construction (N2K, D-011), with an explicit "no write path" invariant.
   The Cyphal control bus is D-011's transmit-allowed exception: add a
   `write_frame` and a read-write open, and update the module doc to carve out
   that exception (keep the listen-only path for N2K).

3. **Hosted wiring.** For a `cyphal_can` bus carrying effectors, build a
   `CyphalActuatorBackend` from the manifest (per-effector CyphalEffector
   subjects, conn node id, power subject, tolerance), select it as that bus's
   output backend, drive it through the trait with a CanSink writing frames,
   and read feedback via `can::spawn_reader` -> `handle_frame` -> `core.power`
   (reuse the existing PowerStatus intake) plus the command-then-report
   divergence into supervisor health.

4. **vcan rig (`hosted/tests/cyphal_can_rig.rs`), mirroring can_rig.rs.** The
   test plays the actuator node: reads command frames off vcan, sends feedback
   frames back, and asserts the command-then-report result and power reach the
   observable surface (health telemetry). Napkin-scale byte tests run
   everywhere; the vcan end-to-end test skips gracefully where vcan is
   unavailable (the require_vcan pattern).

Kick-off for next session: check out `cyphal-actuator-backend`, then start with
part 1 (the schema DECISIONS entry, then the schema_version bump), verifying at
each step (golden tests, no_std gate). Eventually merge to main directly (no
PR), per the operator.

## Cyphal actuator backend: increment 2c done

2c landed. The three open schema decisions from the handoff went, after
discussion with the operator: (1) the conn node's own bus id lives as `node_id`
on the `cyphal_can` `[[bus]]`, and the power node's voltage subject as `subject`
on the role=power `[[sensor]]`; (2) `report_tolerance` is per effector, in that
effector's physical units (a single bus scalar cannot be right for both a
thruster in newtons and a rudder in radians); (3) effectors and
`[[actuator_node]]` stay mutually exclusive, as on the serial path. All three
are recorded in D-029, written before the code.

What shipped:

- Schema v4 -> v5. `EffectorEntry.output` is now an `EffectorOutput` sum type,
  `Serial { channel, pwm }` vs `Cyphal { node_id, command_subject,
  feedback_subject, report_tolerance }`, selected by the effector's bus kind at
  compile; the wrong arm's fields are rejected. `BusEntry.node_id` and
  `SensorEntry.subject` added. coxswain-manifest now depends on coxswain-cyphal
  for the node-id/subject-id wire ranges (single source of truth). New
  `cyphal_effector.rs` golden + one rejection per new rule.
- Backend refactor: `report_tolerance` moved onto `CyphalEffector`;
  `power_subject` is now `Option` (the failsafe matrix already tolerates no
  power link, so a Cyphal actuator bus need not have a power node).
- `hosted/src/can.rs` gained `write_frame`; the module's listen-only invariant
  is carved to the N2K path, with the Cyphal control bus as D-011's
  transmit-allowed exception.
- Hosted wiring: a `CyphalActuatorLink` per `cyphal_can` bus carrying
  effectors, built from the manifest, driven through the output backend trait
  with a `CanSink`, feedback/power read via `can::spawn_reader` ->
  `handle_frame`. Power feeds the existing `PowerStatus` intake; per-effector
  divergence and the supervisor's report-only low-voltage flag surface in the
  published health as `actuation` and supervisor `power` subjects. A new
  `ClaimantClient::subscribe_health` distills both, so a claimant observes them
  without arming.
- vcan rig `hosted/tests/cyphal_can_rig.rs`, mirroring can_rig.rs: byte-level
  napkin tests run everywhere; the end-to-end test plays the actuator/power
  node and asserts command frames, divergence, and voltage reach health, and
  skips gracefully where vcan is unavailable. A no-vcan boot-error test drives
  the real binary through the whole Cyphal boot path (an unopenable control bus
  fails boot).

Not done, deferred honestly: node-local failsafe/heartbeat for a Cyphal
effector (the `[[actuator_node]]` fields) waits for the actuator firmware that
would enforce it (Phase 9); the sensor-only Cyphal read path (a power node on a
bus with no effectors) is not wired, since nothing needs it yet.
