## Phase 1: contract design

Representation choices for coxswain-contract, made against the manifest
schema v0.2 and the Fossen 3-DOF model:

- f64 for all physical quantities. Geodetic position does not survive f32,
  and the H753 has a double-precision FPU, so there is no case for mixing
  widths.
- Vessel state carries geodetic position (WGS84, radians) plus body-frame
  velocities (surge, sway, yaw rate). Covariance is 6x6 over
  [n, e, psi, u, v, r] in the local NED tangent frame: estimation happens
  in the tangent frame, reporting in geodetic.
- The contract crate stays dependency-free apart from optional serde.
  nalgebra lives in estimator/guidance; they convert at the boundary. The
  cost is a few from_column_slice calls, the gain is a contract that never
  moves because a math dependency did.
- Identities are integer newtypes (ClaimantId, SensorId, u16). Names map to
  ids at the adapter edge and in the manifest compiler; authority and
  fusion logic never parse strings.
- Actuator command is the generalized force demand (tau: surge, sway
  forces, yaw moment). Thrust allocation to physical actuators is post-MVP
  per D-021, so the contract does not guess its shape.
- Fixed-capacity BoundedList (hand-rolled, ~80 lines) instead of a heapless
  dependency: only config data needs it, and equality/serde semantics
  (compare and serialize the live prefix only) are easy to get subtly wrong
  in a generic dep wrapper anyway.
- Setpoint enum is deliberately minimal (Idle, HeadingSpeed, StationKeep);
  Phase 4 grows it with path following. Additions are review points.

Candidate for DECISIONS.md: the representation bundle (f64, frames,
dep-free contract, integer identities). Written as D-023.

CI green on GitHub for both the Phase 0 and Phase 1 pushes.

## Phase 2: estimator design

- Measurements carry their own noise std (contract MeasurementKind). The
  source declares what it knows: the simulator its configured noise, a GNSS
  driver its HDOP-derived guess. This defers schema open question 1
  (manifest noise parameters vs priority lists) per D-022 instead of
  answering it by accident. The heading priority list in EstimatorConfig is
  correspondingly unused for now: all licensed heading sensors fuse,
  weighted by reported std.
- EKF over [n, e, psi, u, v, r] in a local NED tangent frame anchored at
  the first accepted GNSS fix. Constant-velocity (constant-twist) process
  until the hydrodynamic prior lands in Phase 3. Euler discretization,
  analytic Jacobian, wrapped heading innovation.
- Process noise constants are documented placeholders until system
  identification (parked task). Replay bounds are deliberately loose.
- Licensing enforced at intake: a measurement fuses only if its sensor id
  is in the matching EstimatorConfig list and carries inner_loop license.
  Rejection is a typed result, not a silent drop.
- Health: Fault until initialized (first fix + first heading), Degraded on
  any stale fused role, else Nominal. Stds reported raw; covariance
  thresholds belong to the supervisor.
- Replay harness lives in coxswain-estimator/tests/ with a hand-rolled
  seeded xorshift RNG (determinism across platforms, no rand dep). JSONL is
  the recorded-log format until real recordings exist. Promote the harness
  to a crate only when another consumer appears.

## Phase 3 (started in parallel): coxswain-model

The model crate depends only on the contract, so it proceeds alongside the
estimator. Diagonal-M Fossen 3-DOF (the schema's param set has no coupling
coefficients, x_g = 0 assumed at the midship origin), SNAME sign
convention, linear damping. API serves both D-020 consumers: nu_dot +
analytic Jacobian for the estimator, RK4 step for the plant. Verification
is closed-form: terminal velocities, exponential time constants, Coriolis
skew-symmetry, passive energy decay, finite-difference Jacobian check, RK4
order check.

## Phase 2 and model results

Model verification numbers: terminal surge relative error 3.36e-4 at eight
time constants, which is exp(-8) itself, so the integrator adds essentially
nothing. RK4 error ratio 18.0 on dt halving (4th order predicts 16).

Estimator replay results (seeded, regression-locked in CI):

- Noise-free straight line: max position error 8 mm, heading 0.002 deg.
- Noisy straight line (GNSS 1 Hz / 2 m, heading 5 Hz / 1 deg, gyro 20 Hz /
  0.01 rad/s, 120 s): position RMSE 2.11 m against 2.83 m raw 2D fix
  noise, heading RMSE 0.29 deg, mean NEES 2.35.
- Turn through the +-pi wrap: heading RMSE 0.32 deg, no 2pi excursions.
- GNSS dropout: position std 4.0 m before the gap, 57 m after 23 s of
  silence, 3.3 m ten seconds after recovery; staleness flag and Degraded
  level flip exactly at max_age.
- An enrichment-licensed biased heading stream: 240/240 rejected, RMSE
  identical to the clean run.

Two honest bound adjustments, commented in the tests rather than tuned
away: position RMSE bound 2.5 m (the mandated 0.5 m/s^2 acceleration
budget runs conservative against a zero-acceleration truth) and NEES band
[1, 12] for the same reason. The filter is conservative, not
overconfident; the upper NEES bound still catches overconfidence
regressions. Process noise mapped as white-acceleration PSD (Q scales with
dt, not dt^2) so covariance growth per second is independent of tick rate.
